# Luppiter Web - 메인터넌스 현황 화면 구조

## 개요

본 문서는 Luppiter Web의 **메인터넌스 현황** 화면의 구조와 처리 흐름을 설명합니다.

---

## 화면 정보

- **URL**: `/view/zab/subMaintenanceStatusInfo`
- **JSP 파일**: `zab/subMaintenanceStatusInfo.jsp`
- **Controller**: `ZabController.java`
- **REST Controller**: `ZabRestController.java`
- **설명**: 관제 대상관리 > 메인터넌스 현황

---

## 화면 구성 요소

### 1. 헤더 영역
- **제목**: 메인터넌스 현황

### 2. 검색 필터 영역 (`search`)
- **메인터넌스 정보**:
  - 메인터넌스ID: `inputMaintenanceId`
  - 호스트 상태: `inputHostSts` (활성, 대기중, 종료, 실패)
  - 시작일/시간: `inMainMtncStartDate`, `inMainMtncStartTime`
  - 종료일/시간: `inMainMtncEndDate`, `inMainMtncEndTime`
- **등록자/장비 정보**:
  - 등록자: `inputMtnRegistNm`
  - 호스트명: `inputHostName`
  - IP: `inputZabbixIp`
  - 관제영역: `ctlArea`
- **레이어 필터**:
  - L1, L2, L3 선택: `commonL1`, `commonL2`, `commonL3`
  - Zone: `zone`
- **담당자 정보**:
  - 운영부서: `manageDept`
  - 메인터넌스명: `inputMtnNm`
  - 사유: `inputReason`
  - 관제부서: `controlDept`
  - 1선담당자: `userName1st` (담당자조회 버튼: `bt-pop-pic1`)
  - 2선담당자: `userName2nd` (담당자조회 버튼: `bt-pop-pic2`)
- **검색/초기화 버튼**: `btnSearch`, `btRefresh`

### 3. 그리드 영역
- **Toast UI Grid**: `grid`
- **페이지 크기 선택**: `slcPageSize` (20, 50, 75, 100, 200, 300, 전체)
- **페이징**: `divPaging`

### 4. 팝업
- **담당자 조회 팝업**: `popup-person-in-charge`

---

## 처리 흐름

### 1단계: 화면 초기화

```
[페이지 로드]
    │
    └─► ZabController.subMaintenanceStatusInfo()
        └─► subMaintenanceStatusInfo.jsp 렌더링

    ▼

[JavaScript 초기화]
    │
    └─► 화면 초기화 함수 호출
        ├─► 그리드 초기화
        └─► 첫 페이지 조회
```

#### 주요 코드 위치
- **컨트롤러**: `ZabController.java:45-56`
- **JSP**: `zab/subMaintenanceStatusInfo.jsp`

### 2단계: 메인터넌스 현황 목록 조회

```
[조회 요청]
    │
    ├─► 검색 버튼 클릭 (btnSearch)
    │   └─► 메인터넌스 현황 목록 조회 함수 호출
    │
    └─► 페이지 로드 시 자동 조회

    ▼

[API 호출]
    │
    └─► POST /api/zab/selectMaintenanceStatusList
        ├─► 파라미터 구성
        │   ├─► 검색 조건 (메인터넌스ID, 호스트 상태, 기간, 등록자, 호스트, IP, 관제영역, 레이어, 담당자 정보 등)
        │   ├─► 페이지 정보 (pageNum, pageSize)
        │   └─► 정렬 정보
        │
        └─► 응답 데이터
            ├─► 메인터넌스 현황 목록 (list)
            ├─► 전체 건수 (totalCount)
            └─► 페이징 정보

    ▼

[데이터 바인딩]
    │
    └─► 그리드 데이터 설정
        ├─► gridObj["grid"].resetData(data.list)
        └─► 페이징 정보 업데이트
```

#### 주요 코드 위치
- **조회 함수**: `subMaintenanceStatusInfo.jsp:메인터넌스 현황 목록 조회 함수`
- **API 호출**: `ZabRestController.java:selectMaintenanceStatusList API`

## 데이터 흐름 다이어그램

```
[사용자 액션]
    │
    └─► 검색 버튼 클릭

    ▼

[API 호출]
    │
    └─► POST /api/zab/selectMaintenanceStatusList
        └─► ZabRestController.selectMaintenanceStatusList()
            └─► ZabService.selectMaintenanceStatusList()

    ▼

[데이터베이스]
    │
    └─► CMON_MAINTENANCE_STATUS 조회
        ├─► 검색 조건 적용
        ├─► 페이징 처리
        └─► 메인터넌스 현황 정보 조회

    ▼

[응답 데이터]
    │
    └─► JSON 응답
        ├─► list: 메인터넌스 현황 목록
        ├─► totalCount: 전체 건수
        └─► paging: 페이징 정보

    ▼

[화면 업데이트]
    │
    └─► 그리드 데이터 바인딩
        ├─► gridObj["grid"].resetData()
        └─► 페이징 정보 업데이트
```
