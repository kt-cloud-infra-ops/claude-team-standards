# Luppiter Web - 대응관리 화면 구조

## 개요

본 문서는 Luppiter Web의 **대응관리** 화면의 구조와 처리 흐름을 설명합니다.

---

## 화면 정보

- **URL**: `/view/ctl/subControlRespMngInfo`
- **JSP 파일**: `ctl/subControlRespMngInfo.jsp`
- **Controller**: `CtlController.java`
- **REST Controller**: `CtlRestController.java`
- **설명**: 관제 운영 관리 > 호스트그룹 대응관리

---

## 화면 구성 요소

### 1. 헤더 영역
- **제목**: 대응관리
- **기능 버튼**:
  - 엑셀다운로드 버튼: `btnExcelDownLoad`

### 2. 검색 필터 영역 (`search`)
- **레이어 필터**:
  - L1, L2, L3 선택: `commonL1`, `commonL2`, `commonL3`
  - Zone: `zone`
  - 관제영역: `ctlArea`
  - 호스트그룹명: `host_group_nm`
  - 대응레벨: `respLevel` (관제/상황대응, 선관제/담당부서연락/상황대응지원, 비관제/상황대응지원)
- **담당자 정보**:
  - 운영부서: `manageDept`
  - 1선담당자: `userName1st` (담당자조회 버튼: `bt-pop-pic1`)
  - 2선담당자: `userName2nd` (담당자조회 버튼: `bt-pop-pic2`)
  - 관제부서: `controlDept`
  - 컨콜그룹: `concallGroup`
  - 대응참고사항: `respInfo`
- **검색/초기화 버튼**: `btnSearch`, `btRefresh`

### 3. 그리드 영역
- **Toast UI Grid**: `grid`
- **페이지 크기 선택**: `slcPageSize` (20, 50, 75, 100, 200, 300, 전체)
- **액션 버튼**:
  - 이력조회: `btnRespHistory`
  - 수정: `btnRespUpdate`
- **페이징**: `divPaging`

### 4. 팝업
- **호스트그룹 대응관리 이력 팝업**: `RespMngHistoryPopup` (`subControlRespMngHistory.jsp`)
- **대응관리 수정 팝업**: `popRespManage` (`subRespMngUpdatePopup.jsp`)

## 처리 흐름

### 1단계: 화면 초기화

```
[페이지 로드]
    │
    └─► CtlController.subControlRespMngInfo()
        └─► subControlRespMngInfo.jsp 렌더링

    ▼

[JavaScript 초기화]
    │
    └─► 화면 초기화 함수 호출
        ├─► 그리드 초기화
        └─► 첫 페이지 조회
```

#### 주요 코드 위치
- **컨트롤러**: `CtlController.java:142-153`
- **JSP**: `ctl/subControlRespMngInfo.jsp`

### 2단계: 대응관리 목록 조회

```
[조회 요청]
    │
    ├─► 검색 버튼 클릭 (btnSearch)
    │   └─► 대응관리 목록 조회 함수 호출
    │
    └─► 페이지 로드 시 자동 조회

    ▼

[API 호출]
    │
    └─► POST /api/ctl/respMngList
        ├─► 파라미터 구성
        │   ├─► 검색 조건 (L1, L2, L3, zone, 관제영역, 호스트그룹명, 대응레벨 등)
        │   ├─► 페이지 정보 (pageNum, pageSize)
        │   └─► 정렬 정보
        │
        └─► 응답 데이터
            ├─► 대응관리 목록 (list)
            ├─► 전체 건수 (totalCount)
            └─► 페이징 정보

    ▼

[데이터 바인딩]
    │
    └─► 그리드 데이터 설정
        ├─► gridObj["grid"].resetData(data.list)
        └─► 페이징 정보 업데이트
```

#### 주요 코드 위치
- **조회 함수**: `subControlRespMngInfo.jsp:대응관리 목록 조회 함수`
- **API 호출**: `CtlRestController.java:respMngList API`

### 3단계: 대응관리 수정

```
[수정 버튼 클릭]
    │
    └─► btnRespUpdate 클릭
        └─► 수정 팝업 오픈
            └─► subRespMngUpdatePopup.jsp

    ▼

[대응관리 수정 처리]
    │
    └─► POST /api/ctl/respMngUpdate
        ├─► 대응관리 정보 수정
        │   ├─► 대응레벨
        │   ├─► 운영부서
        │   ├─► 1선담당자, 2선담당자
        │   ├─► 관제부서
        │   ├─► 컨콜그룹
        │   └─► 대응참고사항
        │
        └─► 수정 완료
            └─► 목록 새로고침
```

#### 주요 코드 위치
- **수정 처리**: `CtlRestController.java:respMngUpdate API`
- **서비스**: `CtlService.updateRespMngInfo()`

### 4단계: 이력 조회

```
[이력조회 버튼 클릭]
    │
    └─► btnRespHistory 클릭
        └─► 이력조회 팝업 오픈
            └─► subControlRespMngHistory.jsp
                └─► POST /api/ctl/respMngHistory
                    └─► 호스트그룹 대응관리 이력 조회
```

## 데이터 흐름 다이어그램

```
[사용자 액션]
    │
    ├─► 검색 버튼 클릭
    ├─► 수정 버튼 클릭
    └─► 이력조회 버튼 클릭

    ▼

[API 호출]
    │
    ├─► POST /api/ctl/respMngList
    │   └─► CtlRestController.respMngList()
    │       └─► CtlService.selectRespMngList()
    │
    ├─► POST /api/ctl/respMngUpdate
    │   └─► CtlRestController.respMngUpdate()
    │       └─► CtlService.updateRespMngInfo()
    │
    └─► POST /api/ctl/respMngHistory
        └─► CtlRestController.respMngHistory()
            └─► CtlService.selectRespMngHistory()

    ▼

[데이터베이스]
    │
    └─► CMON_RESP_MANAGE_INFO 조회/수정
        ├─► 검색 조건 적용
        ├─► 페이징 처리
        └─► 대응관리 정보 수정

    ▼

[응답 데이터]
    │
    └─► JSON 응답
        ├─► list: 대응관리 목록
        ├─► totalCount: 전체 건수
        └─► paging: 페이징 정보

    ▼

[화면 업데이트]
    │
    └─► 그리드 데이터 바인딩
        ├─► gridObj["grid"].resetData()
        └─► 페이징 정보 업데이트
```
