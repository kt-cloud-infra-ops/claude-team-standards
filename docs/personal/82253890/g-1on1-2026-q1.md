## 텍스트박스 복사용


[1. IDP 목표 실행 현황]

◆ 시스템 아키텍처 설계 역량 (70%)
Observability 연동 프로젝트에서 연동 포인트 디커플링 설계를 진행했습니다.
이벤트/인벤토리/미디어 영역을 인터페이스 기반으로 분리하여 다양한 소스를 플러그인처럼 추가할 수 있는 구조를 설계했고, Strategy/Adapter 패턴을 실무에 적용하면서 확장 가능한 구조를 고민하는 시각이 생겼습니다.

◆ 성능 분석 및 최적화 역량 (50%)
Scheduler 이벤트 취합 시간이 13초→36초로 느려지는 문제를 분석했습니다.
프로시저 실행 계획 분석, 임시 테이블 누적 패턴 발견, 인덱스 누락 식별까지 근본 원인을 추적하는 과정을 경험했습니다.

◆ AI 기반 개발 프로세스 구축 (60%)
Claude Code를 개발 프로세스 전반에 통합하는 방법을 연구 중입니다.
CLAUDE.md, rules, hooks를 통해 재현 가능한 AI 개발 환경을 구축했고, 팀에 전파할 AI 활용 표준 가이드를 만들고 싶습니다.


[2. 성장 과정에서의 배움과 어려움]

◆ 배운 점
  → 아키텍처: "기능 동작" 관점에서 "확장 가능한 구조" 관점으로 변화
  → 성능 분석: 정량적 데이터로 가설 수립→검증하는 방법 습득

◆ 어려운 점
  → 학습과 업무 균형: 실제 업무가 생기면 계획한 학습 시간이 밀림
  → 프론트엔드: JSP 기반 업무라 React 학습과 연결이 안 됨
  ※ 업무와 학습이 병행되는 방안이 가장 좋을 것 같습니다.


[3. 코칭 받고 싶은 주제]

◆ 학습 시간 확보 전략: 업무와 학습을 효율적으로 병행하는 방법
◆ 프론트엔드 기술 선택: 팀 내 기술 스택 방향성, 타 팀 사례
◆ 커리어 방향성: 풀스택 확장 vs 백엔드/인프라 집중


[4. 리더에게 바라는 지원 사항]

◆ K8s 실습 환경: 학습용 클러스터 접근 또는 로컬 환경 구축 가이드
◆ MSA 전환 사례 공유: 타 팀 컨테이너 전환 경험 공유 기회


[5. 다음 분기 성장 계획]

◆ 2월: Observability 연동 완료 + K8s 기초 학습 + 자바 개발 역량 향상
◆ 3월: K8s 심화 (Ingress, Helm) + React 기초 학습
◆ 상반기: AWS SAA 취득
