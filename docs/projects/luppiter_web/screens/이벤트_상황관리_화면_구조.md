# Luppiter Web - 이벤트 상황관리 화면 구조

## 개요

본 문서는 Luppiter Web의 **이벤트 상황관리** 화면의 구조와 처리 흐름을 설명합니다.

---

## 화면 정보

- **URL**: `/view/evt/subEventState`
- **JSP 파일**: `evt/subEventState.jsp`
- **Controller**: `EvtController.java`
- **REST Controller**: `EvtRestController.java`
- **관련 화면**: 
  - 이벤트 상황 관리(팝업): `/view/evt/subEventStateWindowPopup`
  - 이벤트 예외 관리: `/view/evt/subEventExcpState`
  - 이벤트 정제 관리: `/view/evt/subEventRefineState`
  - 이벤트 이력 조회: `/view/evt/subEventHistory`

---

## 화면 구성 요소

### 1. 헤더 영역
- **제목**: 이벤트 상황 관리
- **기능 버튼**:
  - 소리끔/켬 버튼 (`btnSound`)
  - 엑셀다운로드 버튼 (`btnExcelDown`)
- **현재 시간 표시**: `printNowDateTime`

### 2. 실시간 조회 영역
- **실시간 체크박스**: `realCHK` (기본 체크됨)
- **검색필터 체크박스**: `btnKeyword`
- **현재 조회 시간 표시**: `currCheckTime`
- **지금 업데이트 버튼**: `btnReload`
- **컬럼선택 버튼**: 컬럼 표시/숨김 설정

### 3. 검색 필터 영역 (`divSearch`)
- **조회기간**:
  - 조회 타입 선택: `searchType` (이벤트 발생 시간/인지 시간/완료 시간)
  - 시작일/시간: `startDate`, `startTime`
  - 종료일/시간: `endDate`, `endTime`
  - 기간 선택 버튼: 1개월, 2개월, 3개월
- **레이어 필터**:
  - L1, L2, L3 선택: `commonL1`, `commonL2`, `commonL3`
  - Zone(L4): `zone`
- **이벤트 필터**:
  - 이벤트등급: `kwEventLevel` (Critical, Fatal)
  - 이벤트상태: `kwZabbixState` (신규, 지속, 해소)
  - 이벤트대응단계: `kwEventState` (신규, 인지, 조치중)
  - 인시던트: `kwIncidentId`
  - 인지자: `kwStep1User`
  - 조치자: `kwStep2User`
  - 이벤트제목: `kwEventContent`
  - 표준서비스명: `kwStdnm`
  - 단위서비스명: `kwEstdnm`
- **장비 정보 필터**:
  - 호스트명: `kwHostname`
  - 관제영역: `ctlArea`
  - 이벤트 수집 IP: `kwTargetIp`
  - 센터: `kwContainerNm`
  - 층/실: `kwDatacenterNm`
  - 상면정보: `kwRackLocation`
  - 이벤트ID: `kwEventId`
- **검색/초기화 버튼**: `btnSearch`, `btnSearchInit`

### 4. 그리드 영역
- **Toast UI Grid**: `grid`
- **페이지 크기 선택**: `slcPageSize` (20, 50, 75, 100, 200, 300, 전체)
- **액션 버튼**:
  - 인지: `btnPerceive`
  - 이벤트 조치완료: `btnEventEnd`
  - 예외 등록: `btnEventExcp`
  - 인시던트확대: `btnIncident`
  - 정제 등록: `btnEventRefine` (숨김)
- **페이징**: `divPaging`

## 처리 흐름

### 1단계: 화면 초기화

```
[페이지 로드]
    │
    ├─► fnInit() 호출
    │   ├─► fnPrintNowDateTime() - 현재 시간 출력
    │   ├─► fnInitGnbLnb() - 메뉴 초기화
    │   ├─► fnInitGrid() - 그리드 초기화
    │   ├─► fnPagenationList("1") - 첫 페이지 조회
    │   ├─► fnAlarm() - 알림 초기화
    │   └─► fnCommon() - 공통 기능 초기화
    │
    └─► fnAutoReload() - 자동 갱신 설정
```

#### 주요 코드 위치
- **초기화 함수**: `subEventState.jsp:1668-1705`
- **그리드 초기화**: `subEventState.jsp:fnInitGrid()`
- **자동 갱신**: `subEventState.jsp:fnAutoReload()`

### 2단계: 이벤트 목록 조회

```
[조회 요청]
    │
    ├─► 검색 버튼 클릭 (btnSearch)
    │   └─► fnPagenationList() 호출
    │
    ├─► 실시간 자동 조회 (realCHK 체크 시)
    │   └─► setInterval() - 주기적 조회
    │
    └─► 지금 업데이트 버튼 (btnReload)
        └─► fnPagenationList() 호출

    ▼

[API 호출]
    │
    └─► POST /api/evt/list
        ├─► 파라미터 구성
        │   ├─► 검색 조건 (조회기간, 필터 등)
        │   ├─► 페이지 정보 (pageNum, pageSize)
        │   └─► 정렬 정보
        │
        └─► 응답 데이터
            ├─► 이벤트 목록 (list)
            ├─► 전체 건수 (totalCount)
            └─► 페이징 정보

    ▼

[데이터 바인딩]
    │
    └─► 그리드 데이터 설정
        ├─► gridObj["grid"].resetData(data.list)
        ├─► fnSetRowBgcolor() - 행 배경색 설정
        └─► 페이징 정보 업데이트
```

#### 주요 코드 위치
- **조회 함수**: `subEventState.jsp:fnPagenationList()`
- **API 호출**: `subEventState.jsp:fnAjax()`
- **그리드 바인딩**: `subEventState.jsp:gridObj["grid"].resetData()`

### 3단계: 이벤트 처리

```
[이벤트 선택]
    │
    ├─► 그리드 행 클릭/체크
    │   └─► 선택된 이벤트 정보 저장
    │
    └─► 액션 버튼 활성화

    ▼

[이벤트 처리 선택]
    │
    ├─► 인지 처리 (btnPerceive)
    │   └─► POST /api/evt/updateOne/{eventType}
    │       └─► 1차 인지 처리
    │
    ├─► 이벤트 조치완료 (btnEventEnd)
    │   └─► POST /api/evt/updateEventEnd
    │       └─► 이벤트 종료 처리
    │
    ├─► 예외 등록 (btnEventExcp)
    │   └─► 예외 등록 팝업 오픈
    │       └─► POST /api/evt/excp/save
    │
    ├─► 인시던트확대 (btnIncident)
    │   └─► POST /api/evt/createIncident
    │       └─► 인시던트 생성
    │
    └─► 정제 등록 (btnEventRefine)
        └─► 정제 등록 팝업 오픈
            └─► POST /api/evt/refine/save
```

#### 주요 코드 위치
- **인지 처리**: `subEventState.jsp:btnPerceive 클릭 이벤트`
- **조치완료**: `subEventState.jsp:btnEventEnd 클릭 이벤트`
- **예외 등록**: `subEventState.jsp:btnEventExcp 클릭 이벤트`
- **인시던트 생성**: `subEventState.jsp:btnIncident 클릭 이벤트`

### 4단계: 이벤트 상세 조회

```
[그리드 더블클릭]
    │
    └─► fnPopupData() 호출
        │
        └─► POST /api/evt/info
            ├─► eventId 파라미터 전달
            │
            └─► 이벤트 상세 정보 조회
                ├─► 이벤트 기본 정보
                ├─► 장비 정보
                ├─► 대응관리 정보
                └─► 이벤트 이력

    ▼

[팝업 표시]
    │
    └─► 이벤트 상세 팝업 오픈
        └─► 상세 정보 표시
```

#### 주요 코드 위치
- **상세 조회**: `subEventState.jsp:fnPopupData()`
- **팝업 오픈**: `subEventState.jsp:그리드 더블클릭 이벤트`
