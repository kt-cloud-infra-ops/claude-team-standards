# Luppiter Web - 사용자 관리 화면 구조

## 개요

본 문서는 Luppiter Web의 **사용자 관리** 화면의 구조와 처리 흐름을 설명합니다.

---

## 화면 정보

- **URL**: `/view/ctl/subControlMember`
- **JSP 파일**: `ctl/subControlMember.jsp`
- **Controller**: `CtlController.java`
- **REST Controller**: `CtlRestController.java`
- **설명**: 관제 운영 관리 > 사용자 관리

---

## 화면 구성 요소

### 1. 헤더 영역
- **제목**: 사용자 관리

### 2. 검색 필터 영역 (`search`)
- **사용권한**: `userGroup` (전체, 관제담당자, 관제자, 운영담당자, 사용자)
- **검색어**:
  - 검색 타입 선택: `searchType` (사용자ID, 사용자명, 소속(조직))
  - 검색어 입력: `keyword`
- **상태**: 상태 선택
- **검색/초기화 버튼**: 검색 버튼, 초기화 버튼

### 3. 그리드 영역
- **Toast UI Grid**: `grid`
- **페이지 크기 선택**: `slcPageSize`
- **액션 버튼**:
  - 등록: 사용자 등록 버튼
  - 수정: 사용자 수정 버튼
  - 삭제: 사용자 삭제 버튼
- **페이징**: `divPaging`

---

## 처리 흐름

### 1단계: 화면 초기화

```
[페이지 로드]
    │
    └─► CtlController.subControlMember()
        └─► subControlMember.jsp 렌더링

    ▼

[JavaScript 초기화]
    │
    └─► 화면 초기화 함수 호출
        ├─► 그리드 초기화
        └─► 첫 페이지 조회
```

#### 주요 코드 위치
- **컨트롤러**: `CtlController.java:65-76`
- **JSP**: `ctl/subControlMember.jsp`

### 2단계: 사용자 목록 조회

```
[조회 요청]
    │
    ├─► 검색 버튼 클릭
    │   └─► 사용자 목록 조회 함수 호출
    │
    └─► 페이지 로드 시 자동 조회

    ▼

[API 호출]
    │
    └─► POST /api/ctl/list
        ├─► 파라미터 구성
        │   ├─► 검색 조건 (사용권한, 검색타입, 검색어, 상태)
        │   ├─► 페이지 정보 (pageNum, pageSize)
        │   └─► 정렬 정보
        │
        └─► 응답 데이터
            ├─► 사용자 목록 (list)
            ├─► 전체 건수 (totalCount)
            └─► 페이징 정보

    ▼

[데이터 바인딩]
    │
    └─► 그리드 데이터 설정
        ├─► gridObj["grid"].resetData(data.list)
        └─► 페이징 정보 업데이트
```

#### 주요 코드 위치
- **조회 함수**: `subControlMember.jsp:사용자 목록 조회 함수`
- **API 호출**: `CtlRestController.java:99-130`

### 3단계: 사용자 등록/수정/삭제

```
[사용자 등록]
    │
    └─► 등록 버튼 클릭
        └─► 사용자 등록 팝업 오픈
            └─► POST /api/ctl/add
                └─► 사용자 등록 처리

    ▼

[사용자 수정]
    │
    └─► 수정 버튼 클릭
        └─► 사용자 수정 팝업 오픈
            └─► POST /api/ctl/update
                └─► 사용자 수정 처리

    ▼

[사용자 삭제]
    │
    └─► 삭제 버튼 클릭
        └─► 삭제 확인
            └─► POST /api/ctl/delete
                └─► 사용자 삭제 처리
```

#### 주요 코드 위치
- **등록 처리**: `CtlRestController.java:add API`
- **수정 처리**: `CtlRestController.java:update API`
- **삭제 처리**: `CtlRestController.java:delete API`

## 데이터 흐름 다이어그램

```
[사용자 액션]
    │
    ├─► 검색 버튼 클릭
    ├─► 사용자 등록 버튼 클릭
    ├─► 사용자 수정 버튼 클릭
    └─► 사용자 삭제 버튼 클릭

    ▼

[API 호출]
    │
    ├─► POST /api/ctl/list
    │   └─► CtlRestController.list()
    │       └─► CtlService.selectCtlList()
    │
    ├─► POST /api/ctl/add
    │   └─► CtlRestController.add()
    │       └─► CtlService.insertCtl()
    │
    ├─► POST /api/ctl/update
    │   └─► CtlRestController.update()
    │       └─► CtlService.updateCtl()
    │
    └─► POST /api/ctl/delete
        └─► CtlRestController.delete()
            └─► CtlService.deleteCtl()

    ▼

[데이터베이스]
    │
    └─► CMON_USER_INFO 조회/수정
        ├─► 검색 조건 적용
        ├─► 페이징 처리
        └─► 사용자 정보 저장/수정/삭제

    ▼

[응답 데이터]
    │
    └─► JSON 응답
        ├─► list: 사용자 목록
        ├─► totalCount: 전체 건수
        └─► paging: 페이징 정보

    ▼

[화면 업데이트]
    │
    └─► 그리드 데이터 바인딩
        ├─► gridObj["grid"].resetData()
        └─► 페이징 정보 업데이트
```
