# Luppiter Web - 설비권한그룹 사용자관리 화면 구조

## 개요

본 문서는 Luppiter Web의 **설비권한그룹 사용자관리** 화면의 구조와 처리 흐름을 설명합니다.

---

## 화면 정보

- **URL**: `/view/ctl/subControlGroup`
- **JSP 파일**: `ctl/subControlGroup.jsp`
- **Controller**: `CtlController.java`
- **REST Controller**: `CtlRestController.java`
- **설명**: 관제 운영 관리 > 설비권한그룹 사용자관리

---

## 화면 구성 요소

### 1. 헤더 영역
- **제목**: 설비권한그룹 사용자관리

### 2. 왼쪽 패널 (`panel`, width:30%)
- **제목**: 설비권한그룹
- **Toast UI Grid**: `gridGroup`

- **제목**: 설비권한그룹 사용자
- **Toast UI Grid**: `gridUser`
- **액션 버튼**:
  - 사용자 추가: 사용자 추가 버튼
  - 사용자 삭제: 사용자 삭제 버튼
- **페이징**: `divPagingUser`

### 4. 팝업
- **설비권한그룹 사용자 이력 팝업**: `subControlGroupUserHistoryPopup.jsp`

## 처리 흐름

### 1단계: 화면 초기화

```
[페이지 로드]
    │
    └─► CtlController.subControlGroup()
        └─► subControlGroup.jsp 렌더링

    ▼

[JavaScript 초기화]
    │
    └─► 화면 초기화 함수 호출
        ├─► 설비권한그룹 그리드 초기화
        ├─► 설비권한그룹 사용자 그리드 초기화
        └─► 첫 페이지 조회
```

#### 주요 코드 위치
- **컨트롤러**: `CtlController.java:103-114`
- **JSP**: `ctl/subControlGroup.jsp`

### 2단계: 설비권한그룹 목록 조회

```
[조회 요청]
    │
    ├─► 조회 버튼 클릭 (bt-pop-usrgrp-inq)
    │   └─► 설비권한그룹 목록 조회 함수 호출
    │
    └─► 페이지 로드 시 자동 조회

    ▼

[API 호출]
    │
    └─► POST /api/ctl/groupList
        ├─► 파라미터 구성
        │   └─► 페이지 정보 (pageNum, pageSize)
        │
        └─► 응답 데이터
            ├─► 설비권한그룹 목록 (list)
            ├─► 전체 건수 (totalCount)
            └─► 페이징 정보

    ▼

[데이터 바인딩]
    │
    └─► 그리드 데이터 설정
        └─► gridGroup.resetData(data.list)
```

#### 주요 코드 위치
- **조회 함수**: `subControlGroup.jsp:설비권한그룹 목록 조회 함수`
- **API 호출**: `CtlRestController.java:groupList API`

### 3단계: 설비권한그룹 사용자 목록 조회

```
[설비권한그룹 선택]
    │
    └─► 그리드 행 클릭
        └─► 선택된 그룹의 사용자 목록 조회

    ▼

[API 호출]
    │
    └─► POST /api/ctl/groupUserList
        ├─► 파라미터 구성
        │   ├─► 그룹 ID
        │   └─► 페이지 정보 (pageNum, pageSize)
        │
        └─► 응답 데이터
            ├─► 설비권한그룹 사용자 목록 (list)
            ├─► 전체 건수 (totalCount)
            └─► 페이징 정보

    ▼

[데이터 바인딩]
    │
    └─► 그리드 데이터 설정
        └─► gridUser.resetData(data.list)
```

### 4단계: 설비권한그룹 관리

```
[그룹 등록]
    │
    └─► 등록 버튼 클릭 (bt-pop-grp-reg)
        └─► 그룹 등록 팝업 오픈
            └─► POST /api/ctl/insertGroup
                └─► 설비권한그룹 등록

    ▼

[그룹 삭제]
    │
    └─► 삭제 버튼 클릭 (bt-grp-del)
        └─► 삭제 확인
            └─► POST /api/ctl/deleteGroup
                └─► 설비권한그룹 삭제

    ▼

[그룹 수정]
    │
    └─► 그리드 행의 수정 버튼 클릭
        └─► POST /api/ctl/updateGroup
            └─► 설비권한그룹 수정
```

### 5단계: 설비권한그룹 사용자 관리

```
[사용자 추가]
    │
    └─► 사용자 추가 버튼 클릭
        └─► 사용자 추가 팝업 오픈
            └─► POST /api/ctl/groupUserAdd
                └─► 설비권한그룹 사용자 추가

    ▼

[사용자 삭제]
    │
    └─► 사용자 삭제 버튼 클릭
        └─► 삭제 확인
            └─► POST /api/ctl/groupUserDelete
                └─► 설비권한그룹 사용자 삭제
```

## 데이터 흐름 다이어그램

```
[사용자 액션]
    │
    ├─► 조회 버튼 클릭
    ├─► 그룹 등록/수정/삭제 버튼 클릭
    ├─► 그룹 행 클릭 (사용자 목록 조회)
    └─► 사용자 추가/삭제 버튼 클릭

    ▼

[API 호출]
    │
    ├─► POST /api/ctl/groupList
    │   └─► 설비권한그룹 목록
    │
    ├─► POST /api/ctl/groupUserList
    │   └─► 설비권한그룹 사용자 목록
    │
    ├─► POST /api/ctl/insertGroup
    │   └─► 설비권한그룹 등록
    │
    └─► POST /api/ctl/groupUserAdd
        └─► 설비권한그룹 사용자 추가

    ▼

[데이터베이스]
    │
    └─► CMON_GROUP_INFO, CMON_GROUP_USER 조회/수정
        ├─► 검색 조건 적용
        ├─► 페이징 처리
        └─► 그룹/사용자 정보 저장/수정/삭제

    ▼

[응답 데이터]
    │
    └─► JSON 응답
        ├─► list: 그룹/사용자 목록
        ├─► totalCount: 전체 건수
        └─► paging: 페이징 정보

    ▼

[화면 업데이트]
    │
    └─► 그리드 데이터 바인딩
        ├─► gridGroup.resetData()
        └─► gridUser.resetData()
```
