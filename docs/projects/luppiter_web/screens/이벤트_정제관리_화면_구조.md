# Luppiter Web - 이벤트 정제 관리 화면 구조

## 개요

본 문서는 Luppiter Web의 **이벤트 정제 관리** 화면의 구조와 처리 흐름을 설명합니다.

---

## 화면 정보

- **URL**: `/view/evt/subEventRefineState`
- **JSP 파일**: `evt/subEventRefineState.jsp`
- **Controller**: `EvtController.java`
- **REST Controller**: `EvtRefineRestController.java`
- **설명**: 이벤트 정제 관리 화면

---

## 화면 구성 요소

### 1. 헤더 영역
- **제목**: 이벤트 정제 관리

### 2. 검색 필터 영역 (`search`)
- **정제기간**:
  - 시작일: `inSearchExcpStartDate`
  - 종료일: `inSearchExcpEndDate`
- **구분 유형**:
  - 정제 상태 선택: `kwSearchRefineType` (전체, 정제검토, 정제중, 반려, 정제완료)
- **검색어**:
  - 검색 타입 선택: `kwSearchType` (정제제목, 이벤트명, 호스트명, IP)
  - 검색어 입력: `kwSearchName`
- **검색/초기화 버튼**: `btSearch`, 초기화 버튼

### 3. 그리드 영역
- **Toast UI Grid**: `grid`
- **페이지 크기 선택**: `slcPageSize` (20, 50, 75, 100)
- **액션 버튼**:
  - 정제 등록: `btnEventRefine`
- **페이징**: `divPaging`

### 4. 팝업
- **정제 수정/이력확인 팝업**: `test-popup1` (`subEventRefineStatePopup.jsp`)
- **이벤트 상세내역 팝업**: `test-popup2` (`subEventRefineStateDetailPopup.jsp`)
- **이벤트 정제 등록 팝업**: `test-refine-popup2` (`subEventStatePopupRefine.jsp`)
- **장비 추가 팝업**: `test-popup2-sub` (`subEventStatePopupDeviceAdd.jsp`)

## 처리 흐름

### 1단계: 화면 초기화

```
[페이지 로드]
    │
    └─► EvtController.subEventRefineState()
        └─► subEventRefineState.jsp 렌더링

    ▼

[JavaScript 초기화]
    │
    └─► 화면 초기화 함수 호출
        ├─► 그리드 초기화
        └─► 첫 페이지 조회
```

#### 주요 코드 위치
- **컨트롤러**: `EvtController.java:45-55`
- **JSP**: `evt/subEventRefineState.jsp`

### 2단계: 이벤트 정제 목록 조회

```
[조회 요청]
    │
    ├─► 검색 버튼 클릭 (btSearch)
    │   └─► 이벤트 정제 목록 조회 함수 호출
    │
    └─► 페이지 로드 시 자동 조회

    ▼

[API 호출]
    │
    └─► POST /api/evt/refine/list
        ├─► 파라미터 구성
        │   ├─► 정제기간 (시작일, 종료일)
        │   ├─► 구분 유형 (정제 상태)
        │   ├─► 검색 조건 (검색타입, 검색어)
        │   ├─► 페이지 정보 (pageNum, pageSize)
        │   └─► 정렬 정보
        │
        └─► 응답 데이터
            ├─► 이벤트 정제 목록 (list)
            ├─► 전체 건수 (totalCount)
            └─► 페이징 정보

    ▼

[데이터 바인딩]
    │
    └─► 그리드 데이터 설정
        ├─► gridObj["grid"].resetData(data.list)
        └─► 페이징 정보 업데이트
```

#### 주요 코드 위치
- **조회 함수**: `subEventRefineState.jsp:이벤트 정제 목록 조회 함수`
- **API 호출**: `subEventRefineState.jsp:fnAjax()`

### 3단계: 이벤트 정제 등록

```
[정제 등록 버튼 클릭]

        │
        └─► 정제 등록 완료
```

## 필요한 데이터 (API 요청)

### 이벤트 정제 목록 조회 파라미터

| 필드명 | 타입 | 설명 | 필수 여부 |
|--------|------|------|-----------|
| `pageNum` | Integer | 페이지 번호 | 필수 |
| `pageSize` | Integer | 페이지 크기 | 필수 |
| `inSearchExcpStartDate` | String | 정제 시작일 | 선택 |
| `inSearchExcpEndDate` | String | 정제 종료일 | 선택 |
| `kwSearchRefineType` | String | 구분 유형 (1:정제검토, 2:정제중, 3:반려, 4:정제완료) | 선택 |
| `kwSearchType` | String | 검색 타입 (title/evtName/hostname/ip) | 선택 |
| `kwSearchName` | String | 검색어 | 선택 |

### 이벤트 정제 등록 파라미터

| 필드명 | 타입 | 설명 | 필수 여부 |
|--------|------|------|-----------|
| `title` | String | 정제 제목 | 필수 |
| `startDate` | String | 정제 시작일 | 필수 |
| `startTime` | String | 정제 시작시간 | 필수 |
| `endDate` | String | 정제 종료일 | 필수 |
| `endTime` | String | 정제 종료시간 | 필수 |
| `reason` | String | 정제 사유 | 필수 |
| `details` | Array | 대상 장비 목록 | 필수 |
| `details[].triggerId` | String | 트리거 ID | 필수 |
| `details[].hostname` | String | 호스트명 | 필수 |
| `details[].ip` | String | IP | 필수 |
