다음 5가지 분석을 **병렬로** 수행하세요:

## 1. 문서 업데이트 분석
이번 세션에서 작업한 내용 중 문서화가 필요한 것이 있는지 확인:
- AGENTS.md/CLAUDE.md 포인터 업데이트 필요 여부
- 하위 프로젝트의 docs 폴더에 추가할 내용
- 새로운 의사결정이 있었다면 `docs/decisions/`에 ADR 작성 제안

## 2. 자동화 패턴 발견
이번 세션에서 반복된 작업이나 자동화할 수 있는 패턴이 있는지 분석:
- 반복적으로 실행한 명령어
- 비슷한 패턴의 코드 수정
- 슬래시 커맨드나 훅으로 만들 수 있는 작업
→ 발견 시 `docs/automations/`에 기록 제안

## 3. 학습 내용 추출
이번 세션에서 새롭게 알게 된 것들:
- 기술적 인사이트
- 프로젝트 구조나 코드베이스에 대한 이해
- 문제 해결 과정에서 배운 것
→ 유용한 내용은 `docs/lessons_learned/`에 기록 제안

## 4. 후속 작업 제안
이번 세션에서 완료하지 못했거나 다음에 해야 할 작업:
- 미완료 TODO 항목
- 발견된 개선점
- 다음 세션에서 이어서 할 작업

## 5. 세션 요약
- 이번 세션에서 완료한 주요 작업 (bullet points)
- 변경된 파일 목록
- 소요된 대략적인 작업 범위

```markdown
## 발견된 자동화 패턴
- ...

## 학습 내용
- ...

## 후속 작업
- [ ] ...
```

위 분석 후, 사용자에게 문서 업데이트 여부를 확인하고 승인 시 실제로 docs 폴더에 파일을 생성하세요.
