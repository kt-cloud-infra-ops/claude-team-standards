# Review Rules Change (공통룰 변경 리뷰)

`.agents/rules/`, `.agents/commands/`, `AGENTS.md` 변경 시 팀 리뷰 프로세스.
코드 머지가 아닌 **팀 규칙 변경**이므로, 변경 영향도와 팀원 동의 여부를 확인한다.

## 실행 절차

1. **변경 파일 식별**: `.agents/rules/`, `.agents/commands/`, `AGENTS.md` 중 수정된 파일 확인
   ```
   git diff --name-only HEAD -- .agents/rules/ .agents/commands/ AGENTS.md
   ```

2. **변경 분류**: 각 변경을 아래 카테고리로 분류
   - **신규 룰 추가**: 새 파일 생성 (팀 전체 영향)
   - **기존 룰 수정**: 동작 변경 (AI 에이전트 응답이 달라짐)
   - **오타/문서 수정**: 동작 변경 없음 (경미)
   - **커맨드 추가/수정**: 슬래시 커맨드 변경

3. **영향도 분석**: 각 변경에 대해
   - **누가 영향받는가**: 전 팀원 / 특정 서비스 담당자 / 본인만
   - **AI 에이전트 동작이 달라지는가**: Yes → 반드시 팀원 확인 필요
   - **기존 워크플로우가 깨지는가**: Yes → CRITICAL 변경

4. **리뷰 리포트 생성**:
   ```markdown
   ## 공통룰 변경 리뷰

   ### 변경 요약
   | 파일 | 분류 | 영향 범위 | 동작 변경 |
   |------|------|----------|----------|
   | ... | ... | ... | Yes/No |

   ### 상세 변경 내용
   (각 파일별 before/after diff 요약)

   ### 리뷰 판정
   - [ ] 경미 (오타/문서) → 바로 커밋 가능
   - [ ] 일반 (기존 룰 보완) → 팀 채널 공유 후 커밋
   - [ ] 중요 (신규 룰/동작 변경) → 팀원 확인 후 커밋
   - [ ] CRITICAL (워크플로우 변경) → 팀 미팅에서 논의
   ```

5. **커밋 메시지 컨벤션**:
   - `rules: <설명>` — 룰 변경
   - `commands: <설명>` — 커맨드 변경
   - 본문에 영향 범위와 리뷰 판정 포함

## 참고

- `.agents/rules/` = **팀 표준** (Git 공유, 전원 영향)
- `~/.claude/rules/` = **개인 설정** (로컬만, 공유 불필요)
- 개인 설정을 팀 표준으로 승격할 때도 이 리뷰 프로세스 사용
